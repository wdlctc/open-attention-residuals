\begin{abstract}
Attention Residuals replace standard additive residual connections with learned softmax attention over previous layer outputs, enabling selective cross-layer routing. We systematically study two overlooked design dimensions. \textbf{(1)~What to route}: we show that attending over per-sublayer outputs (``deltas'') rather than cumulative hidden states is critical---cumulative states exhibit cosine similarity ${\sim}0.95$ between adjacent sources, collapsing the softmax to near-uniform weights. Delta sources reduce WikiText-2 perplexity by 7.5\% over the baseline and 19.2\% over cumulative-state AttnRes at 0.6B scale. \textbf{(2)~Where to route}: we decouple depth routing for the value stream from query/key, since V carries content while Q/K encode positional matching. A full $2{\times}2$ ablation confirms these dimensions are orthogonal and composable---their combination achieves the best results across all configurations. Additionally, Delta AttnRes's additive formulation enables effective fine-tuning: combined with null source initialization, it achieves 29.2\% PPL reduction on Qwen3-0.6B with zero initialization disruption. All improvements require only 0.034\% additional parameters. Code and pretrained weights are available at \url{https://github.com/wdlctc/open-attention-residuals}.
\end{abstract}

\section{Introduction}
%% ============================================================

Residual connections~\citep{he2016deep} are fundamental to training deep transformers~\citep{vaswani2017attention}. The standard update $h_l = h_{l-1} + f_{l-1}(h_{l-1})$ accumulates all sublayer outputs with fixed unit coefficients, providing gradient highways~\citep{veit2016residual} but no selectivity over the depth dimension. Attention Residuals~\citep{kimi2025attention} address this by replacing fixed aggregation with learned softmax attention over previous outputs, achieving improved scaling laws at up to 16B parameters.

Yet the design space of cross-layer routing remains underexplored. We identify two orthogonal dimensions that substantially impact routing effectiveness:

\begin{itemize}
    \item \textbf{What to route} (source representation): Should the model attend over cumulative hidden states $h_i$ or individual sublayer outputs $v_i$ (``deltas'')? We show these are far from equivalent: cumulative states have cosine similarity ${\sim}0.95$ between adjacent sources, collapsing softmax routing to near-uniform. Delta sources avoid this by construction (cos sim $<$0.3), enabling sharp, meaningful cross-layer shortcuts.
    \item \textbf{Where to route} (per-stream routing): Should all information streams (Q, K, V) within each attention sublayer share the same depth-routing decision? We show they should not: Q/K handle positional matching while V carries content, and decoupling their routing provides consistent complementary gains.
\end{itemize}

These two dimensions are \emph{orthogonal and composable}. A systematic $2{\times}2$ ablation---$\{$cumulative, delta$\}$ $\times$ $\{$shared, per-stream V$\}$---confirms that each dimension provides independent gains, and their combination achieves the best results at only 0.034\% additional parameter cost (Figure~\ref{fig:design_space}).

\begin{figure}[t]
\centering
\includegraphics[width=\textwidth]{figures/fig1_delta_vs_cumulative.png}
\caption{\textbf{Two dimensions of cross-layer routing.} (a)~Source redundancy: cumulative states $h_i$ have cosine similarity ${\sim}0.95$ (red), while delta outputs $v_i$ are naturally distinctive ($<$0.3, green). (b)~Routing quality: delta sources achieve higher utilization and sharper weights. (c)~Training loss comparison at 0.6B scale. Across both dimensions, delta sources with per-stream V-routing (Delta-V) achieves the best results.}
\label{fig:design_space}
\end{figure}

Our contributions are:
\begin{enumerate}
    \item \textbf{What to route}: We identify the source redundancy problem and show that delta sources resolve it, achieving 7.5--19.2\% PPL reduction across scales (\S\ref{sec:what}, \S\ref{sec:experiments}).
    \item \textbf{Where to route}: We introduce per-stream V-routing, providing orthogonal gains on top of both source types. The $2{\times}2$ ablation confirms composability (\S\ref{sec:where}, \S\ref{sec:ablation}).
    \item \textbf{Practical deployment}: Delta AttnRes's additive formulation enables fine-tuning of pretrained models with zero initialization disruption (\S\ref{sec:finetune}).
\end{enumerate}
We release code and pretrained weights at \url{https://github.com/wdlctc/open-attention-residuals}.

\section{Two Dimensions of Cross-Layer Routing}
\label{sec:method}
%% ============================================================

\subsection{Preliminaries: Attention Residuals}

A Pre-Norm transformer~\citep{vaswani2017attention,xiong2020prenorm} updates the hidden state as $h_l = h_{l-1} + f_l(h_{l-1})$, accumulating sublayer outputs $v_i = f_i(h_i)$ with fixed unit coefficients. Attention Residuals~\citep{kimi2025attention} replace this with a learned weighted sum:
\begin{equation}
    \hat{h}_l = \sum_{i=0}^{l-1} \alpha_{i \to l} \cdot s_i, \quad
    \alpha_{i \to l} = \frac{\exp(w_l^\top \mathrm{RMSNorm}(s_i))}{\sum_{j} \exp(w_l^\top \mathrm{RMSNorm}(s_j))}
    \label{eq:attnres}
\end{equation}
where $s_i$ are the sources, $w_l \in \mathbb{R}^d$ is a learned query (zero-initialized), and $\hat{h}_l$ feeds into the next sublayer. Two questions arise: \emph{what should $s_i$ be?} and \emph{should all streams share the same $\hat{h}_l$?}

\subsection{Dimension 1: What to Route --- Delta Sources}
\label{sec:what}

\begin{figure}[t]
\centering
\includegraphics[width=\textwidth]{figures/fig1_architecture.png}
\caption{\textbf{Architecture comparison.} (a)~Standard residual: uniform accumulation. (b)~Cumulative-state AttnRes: attends over $h_0, h_1, \ldots$ but adjacent states have cos sim ${\sim}0.95$, degrading softmax routing. (c)~\textbf{Delta AttnRes}: attends over individual sublayer outputs $v_0, v_1, \ldots$ (cos sim $<$0.3), enabling sharp routing, and \emph{adds} the result to the residual stream.}
\label{fig:architecture}
\end{figure}

We consider two source representations for $s_i$ in Eq.~\ref{eq:attnres}:

\paragraph{Cumulative sources.} $s_i = h_i = h_0 + \sum_{j=1}^{i} v_j$---the full hidden state after sublayer $i$. Intuitive but highly redundant: since $h_{i+1} = h_i + v_i$ with $\|v_i\| \ll \|h_i\|$, adjacent sources are nearly identical.

\paragraph{Delta sources.} $s_i = v_i = f_i(h_i)$---the incremental output of sublayer $i$. Structurally diverse because attention and MLP outputs occupy different subspaces, and outputs from different depths capture different abstraction levels.

\begin{proposition}[Source Redundancy]
For sublayer outputs with bounded norm $\|v_i\| \leq \epsilon \|h_i\|$ ($\epsilon \ll 1$):
\begin{equation}
    \mathrm{cos}(h_i, h_{i+1}) \geq 1 - O(\epsilon^2)
\end{equation}
Thus softmax logits $w^\top \mathrm{RMSNorm}(h_i) \approx w^\top \mathrm{RMSNorm}(h_{i+1})$, and the routing distribution approaches uniform.
\end{proposition}

Delta AttnRes uses \emph{additive routing}: $h = \tilde{h} + \sum \alpha_i v_i$, adding selected delta information to the existing residual stream, rather than replacing it. This has an important consequence for initialization: at init ($w_l = 0$), uniform attention over small deltas produces a mild perturbation, whereas uniform attention over cumulative states produces a destructive replacement (\S\ref{sec:finetune}).

\subsection{Dimension 2: Where to Route --- Per-Stream V-Routing}
\label{sec:where}

In standard Attention Residuals, a single depth-routed input $\hat{h}_l$ is shared by the Q, K, and V projections within each attention sublayer. But Q/K and V serve fundamentally different roles:
\begin{itemize}
    \item \textbf{Q/K}: positional and relational matching (``where to attend in the sequence'')
    \item \textbf{V}: content propagation (``what information to retrieve'')
\end{itemize}
These functions may prefer different cross-layer sources. MUDDFormer~\citep{wang2024muddformer} confirms this for MLP-generated weights: ablating V-stream connections causes the largest degradation ($-$0.28 PPL) among all streams.

We propose \textbf{per-stream V-routing}: two independent depth queries, one for Q/K and one for V:
\begin{align}
    \hat{h}_l^{qk} &= \tilde{h}_l + \sum_{i} \alpha_{i \to l}^{qk} \cdot s_i, &
    \alpha_{i \to l}^{qk} &= \mathrm{softmax}_i(w_l^{qk \top} \cdot \mathrm{Norm}(s_i)) \label{eq:qk_routing} \\
    \hat{h}_l^{v} &= \tilde{h}_l + \sum_{i} \alpha_{i \to l}^{v} \cdot s_i, &
    \alpha_{i \to l}^{v} &= \mathrm{softmax}_i(w_l^{v \top} \cdot \mathrm{Norm}(s_i)) \label{eq:v_routing}
\end{align}
The attention sublayer then computes $Q, K$ from $\hat{h}_l^{qk}$ and $V$ from $\hat{h}_l^{v}$. The MLP sublayer uses a single routing query as before. This adds only one query vector $w_l^v \in \mathbb{R}^d$ and one RMSNorm per layer.

Crucially, per-stream V-routing is \emph{orthogonal to the source representation choice}: it can be applied to both cumulative ($s_i = h_i$) and delta ($s_i = v_i$) sources, yielding a $2 \times 2$ design space.

\paragraph{Contrast with MUDDFormer.} MUDDFormer~\citep{wang2024muddformer} decouples all four streams (Q, K, V, residual) using unconstrained MLP-generated weights with 0.23\% overhead. We decouple only V (the most impactful stream per their ablation) within the softmax framework, achieving gains with $10\times$ fewer parameters (0.011\%) and interpretable routing weights.

\subsection{Delta-V AttnRes: Combining Both Dimensions}
\label{sec:combined}

The full method, \textbf{Delta-V AttnRes}, combines delta sources with per-stream V-routing. Figure~\ref{fig:code} shows the complete implementation.

\begin{figure}[t]
\begin{lstlisting}[style=python]
def depth_route(sources: list[Tensor], residual: Tensor,
                proj: Linear, norm: RMSNorm) -> Tensor:
    """Softmax attention over depth sources, added to residual."""
    V = torch.stack(sources)                       # [N, B, T, D]
    K = norm(V)
    logits = einsum('d, n b t d -> n b t', proj.weight.squeeze(), K)
    return residual + einsum('n b t, n b t d -> b t d', logits.softmax(0), V)

def forward(self, deltas, hidden_states):
    # --- Attention sublayer: per-stream routing ---
    h_qk = depth_route(deltas, hidden_states, self.qk_proj, self.qk_norm)
    h_v  = depth_route(deltas, hidden_states, self.v_proj,  self.v_norm)

    attn_out = self.attn(Q=norm(h_qk), K=norm(h_qk), V=norm(h_v))
    hidden_states = hidden_states + attn_out
    deltas.append(attn_out)                        # store attention delta

    # --- MLP sublayer: single routing ---
    h = depth_route(deltas, hidden_states, self.mlp_proj, self.mlp_norm)

    mlp_out = self.mlp(norm(h))
    hidden_states = hidden_states + mlp_out
    deltas.append(mlp_out)                         # store MLP delta

    return deltas, hidden_states
\end{lstlisting}
\caption{\textbf{Delta-V AttnRes pseudocode.} \texttt{depth\_route} computes additive softmax routing over delta sources. The attention sublayer uses two independent routing queries ($w^{qk}$ and $w^v$); the MLP sublayer uses one. Each sublayer appends its output to the delta history.}
\label{fig:code}
\end{figure}

\paragraph{Overhead.} Delta-V AttnRes adds $3$ query vectors and $3$ RMSNorm layers per transformer layer ($2$ for the attention sublayer's Q/K and V routing, $1$ for MLP). For a 28-layer model with $d{=}1024$: 172K parameters (\textbf{0.034\%} of Qwen3-0.6B). Compute overhead is ${\sim}0.6\%$ of sublayer cost. Memory overhead from storing $2L$ delta tensors (${\sim}$470MB at $T{=}4096$, $d{=}1024$) is the dominant cost and may require gradient checkpointing at larger scales.

\subsection{Setup}

We train Qwen3-architecture~\citep{qwen2025qwen3} models from scratch on FineWeb-Edu~\citep{penedo2024fineweb}:
\begin{itemize}
    \item \textbf{Scales}: 100M ($d{=}512$, $L{=}12$), 0.6B ($d{=}1024$, $L{=}28$), and 1.7B ($d{=}2048$, $L{=}28$)
    \item \textbf{Training}: AdamW~\citep{loshchilov2019adamw} ($\beta_1{=}0.9$, $\beta_2{=}0.95$, wd${=}0.1$), cosine LR with warmup. 100M: lr $6{\times}10^{-4}$; 0.6B/1.7B: lr $3{\times}10^{-4}$
    \item \textbf{Budget}: 20K steps at 100M/0.6B; 20--50K at 1.7B

\subsection{Fine-Tuning Pretrained Models}
\label{sec:finetune}

A practical deployment path is fine-tuning Attention Residuals into existing models. Prior work~\citep{pagliardini2024denseformer} found this fails because ``the model commits early to a loss landscape valley that does not use cross-layer weights.'' We show that \emph{additive delta routing} resolves this.

\paragraph{The initialization problem.} Cumulative-state AttnRes uses \emph{replacement routing}: $h = \sum \alpha_i h_i$, which at init ($w_l = 0$) returns the uniform average of all cumulative states---a fundamentally different signal than the pretrained residual. This causes loss to spike from 2.06 to 3.64 (77\% disruption).

\paragraph{Delta AttnRes solution.} Additive routing $h = \tilde{h} + \sum \alpha_i v_i$ preserves the residual stream by default. At init, uniform attention over small deltas ($\|v_i\| \ll \|h_i\|$) produces a mild perturbation. Adding a zero-initialized \emph{null source} $v_\emptyset = \mathbf{0}$ provides an explicit identity path, eliminating disruption entirely.

\subsection{Training Dynamics}

At initialization ($w_l = 0$), softmax gives uniform weights and the model averages all sources. Queries specialize progressively, transitioning smoothly from uniform to selective routing without loss spikes. Block variants converge faster (fewer, more distinctive sources) but Delta variants generalize better (finer-grained routing).

\section{Related Work}
%% ============================================================

\paragraph{Cross-Layer Connections.} DenseNet~\citep{huang2017densely} concatenates all previous outputs ($O(L^2 d)$ memory). DenseFormer~\citep{pagliardini2024denseformer} uses static Depth-Weighted Average weights. Hyper-Connections~\citep{zhu2024hyperconnections} widen the residual stream to $n$ channels; mHC~\citep{deepseek2025mhc} constrains mixing to the doubly stochastic manifold. MUDDFormer~\citep{wang2024muddformer} generates dynamic, per-stream weights via MLPs, showing 2.8B matches 6.9B Pythia. Attention Residuals~\citep{kimi2025attention} use softmax attention over depth, scaling to 16B. We identify two overlooked dimensions in this design space: source representation and per-stream routing.

\paragraph{Residual Stream Analysis.} \citet{veit2016residual} view residual networks as ensembles; \citet{elhage2021mathematical} formalize the residual stream. Residual stream duality~\citep{zhang2026duality} shows depth attention is dual to sequence-axis sliding-window attention. Our source redundancy analysis complements these views.

\paragraph{Gating and Routing.} GLU~\citep{shazeer2020glu}, ReZero~\citep{bachlechner2020rezero}, and MoE~\citep{shazeer2017moe,fedus2022switch} route along the width dimension; we route along \emph{depth}. Deep Delta Learning~\citep{zhang2026ddl} uses Householder reflections for geometric depth control; we use attention-based selection.

\section{Conclusion}
%% ============================================================

We identify two orthogonal design dimensions for cross-layer routing in transformers:
\begin{enumerate}
    \item \textbf{Delta sources} resolve source redundancy (cos sim 0.95 $\to$ $<$0.3), enabling 7.5--19.2\% PPL reduction.
    \item \textbf{Per-stream V-routing} provides complementary gains across both source types; the combination is optimal.
    \item \textbf{Additive formulation} enables fine-tuning with zero initialization disruption.
\end{enumerate}
All at 0.034\% parameter overhead. Code and weights: \url{https://github.com/wdlctc/open-attention-residuals}.

\section{Block Attention Residuals Implementation}
\label{app:block_attnres}
%% ============================================================

We reproduce the PyTorch pseudocode for Block AttnRes from~\citet{kimi2025attention} in Appendix for comparison.

\begin{figure}[h]
\begin{lstlisting}[style=python]
def block_attn_res(blocks, partial_block, proj, norm):
    V = torch.stack(blocks + [partial_block])
    K = norm(V)
    logits = einsum('d, n b t d -> n b t', proj.weight.squeeze(), K)
    return einsum('n b t, n b t d -> b t d', logits.softmax(0), V)
\end{lstlisting}
\caption{Block AttnRes~\citep{kimi2025attention}: replacement routing over cumulative block states.}
\end{figure}

Key differences from Delta-V AttnRes: (1)~sources are cumulative states (high redundancy) vs.\ sublayer deltas (diverse); (2)~routing \emph{replaces} the residual stream vs.\ \emph{adds} to it; (3)~single shared routing vs.\ per-stream V-routing; (4)~$O(N)$ block-boundary sources vs.\ $O(2L)$ per-sublayer sources.
